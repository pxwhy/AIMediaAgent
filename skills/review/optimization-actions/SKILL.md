# 优化动作

## 使用时机

当用户需要把复盘结论转成下一步执行计划、内容实验和观察指标时使用。

## 输入要求

需要尽量提供：
- 复盘结论
- 历史作品表现
- 当前账号问题
- 可执行周期，例如下一周或下一批内容

## 执行步骤

1. 从复盘结论中提炼最重要的 3 到 5 个问题或机会。
2. 为每个问题设计一个具体动作。
3. 将动作映射到可观察指标。
4. 给出执行优先级和复盘周期。
5. 明确哪些动作可以马上做，哪些需要实验验证。

动作要求：
- 具体
- 可验证
- 能在下一批内容中执行
- 尽量对应某个问题或机会
- 不要只给原则性建议

优先从这些方向给建议：
- 标题优化
- 开头优化
- 内容结构优化
- 选题聚焦
- 发布时间或频率
- 评论互动引导
- 系列化内容设计

## 输出格式

请按以下结构输出：

1. 立即执行动作
2. 下一批内容实验
3. 观察指标
4. 执行优先级
5. 复盘周期

## 注意事项

- 不要输出无法衡量的建议。
- 每个动作都要对应一个问题或机会。
- 动作数量不要过多，优先保证能执行。
