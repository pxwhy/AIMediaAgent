# AI 项目上下文

## 项目定位

AIMediaAgent 是一个本地运行的内容运营工作台，面向“账号管理 -> 内容采集 -> 智能筛选 -> 发布执行 -> 账号复盘/肖像”的闭环。

当前形态不是单纯的 AI Demo，而是一个结合本地浏览器自动化、模型调用、人工审核和运营台界面的混合系统。

## 当前真实能力

- 账号管理：支持头条号、小红书登录会话接入，账号按 `platform + uid` 去重。
- 账号操作：支持单个删除、前端批量删除、头条号作品同步。
- 作品能力：支持同步标题、正文、发布时间等作品信息，并在前端弹框查看正文。
- 采集能力：支持来源/分类预览、导入素材库、素材去重。
- 智能筛选：支持基于账号复盘、账号肖像、挑选 Agent 对素材做智能选择。
- 模型配置：支持 DeepSeek 和 OpenAI-compatible 配置、测试、默认模型切换。
- Agent 管理：支持复盘 Agent、肖像 Agent、素材挑选 Agent，允许绑定本地 Skills。
- Skills 管理：支持扫描固定目录下的 `SKILL.md` 并在 Agent 中绑定使用。
- 发布能力：支持头条号和小红书的打开编辑器、自动发布、诊断详情查看。
- 数据分析：支持账号复盘、账号肖像生成、历史记录查看与删除。

## 技术形态

```text
FastAPI API
Vue3 H5 管理端
Python Worker / 子进程任务
Playwright 平台自动化
SQLite 本地数据
Tauri 桌面壳（规划中，当前主要通过脚本启动）
```

## 当前阶段

项目已经完成 MVP 骨架，进入“本地运营工作台”阶段：

- 后端已有核心数据模型、接口、服务分层。
- 前端已有完整主工作台，覆盖账号、采集、发布、模型、Agent、Skills、复盘、肖像、智能筛选。
- 头条号作品同步已经接入。
- 小红书、头条号自动发布已接到统一发布任务流。
- 详情查看、历史查看、诊断查看正在统一成弹框交互。

## 核心链路

```text
账号接入
  -> 同步作品 / 手工采集内容
  -> 素材进入 RawContent
  -> 智能筛选挑出可用素材
  -> 生成或创建发布草稿
  -> 创建发布任务
  -> 平台执行发布
  -> 生成发布结果和诊断信息
  -> 基于作品生成账号复盘
  -> 基于复盘生成账号肖像
  -> 再反哺智能筛选和选题判断
```

## 项目原则

- H5 是主操作界面，桌面端只负责承载本地服务和窗口。
- 账号登录态只能由账号模块维护，不能外泄给模型或前端日志。
- Agent 只做判断、总结、建议，不直接控制浏览器发布。
- Publisher 只执行发布任务，不做内容决策。
- Collector 只负责采集与入库，不承担发布判断。
- 所有自动化都必须保留失败状态、日志和诊断信息。
