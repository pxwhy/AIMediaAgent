# AI 编码规则

## 协作规则

- 新需求编码前，先和用户确认设计方案，不直接动代码。
- 默认使用中文沟通、中文注释。
- 业务代码文件顶部必须写“实现逻辑”注释。
- 修改功能时同步更新相关文档。

## 代码风格

### 命名

- Python 使用 `snake_case`。
- TypeScript / Vue 变量和函数使用 `camelCase`。
- 组件名、类型名、类名使用 `PascalCase`。
- 常量命名要语义化，避免 `tmp`、`foo`、`data2` 这类无意义名字。

### 文件职责

- 一个文件只解决一类问题。
- 路由层只处理入参、出参、错误码，不堆业务逻辑。
- 业务逻辑放 `services/`。
- 数据模型放 `models/`。
- 请求封装和前端类型统一放 `src/api/`。
- 新前端业务模块按职责建立目录，优先拆成 `index`、`add`、`edit` 等入口，避免列表、创建、编辑长期堆在一个大组件里。

### 函数风格

- 单个函数职责明确，优先短函数。
- 复杂分支优先拆成小函数。
- 减少隐式副作用。
- 不要把平台判断、模型调用、页面状态处理揉成一个超长函数。

### 注释风格

- 顶部“实现逻辑”注释写清楚本文件职责。
- 注释优先解释“为什么这么做”，不是机械解释代码表面含义。
- 删除或重构代码时同步清理过时注释。

## 前端规则

- H5 只通过 API 与后端交互，不直接读写本地文件。
- 查看、详情、编辑优先使用弹框，交互尽量统一。
- 新增操作放在列表工具栏，点击后打开弹框表单，不在表格上方或行内展开大表单。
- 编辑操作优先打开弹框表单，表格行内只保留入口按钮，不承载复杂编辑状态。
- 新模块列表页使用 `index` 入口，新增表单使用 `add` 入口，编辑表单使用 `edit` 入口；弹框只是展示方式，组件仍按职责拆分。
- 弹框中的长内容优先让内容区独立滚动，标题、模型信息、关闭操作等头部信息优先固定。
- 素材详情、复盘详情、账号肖像、发布诊断等场景，不再优先使用页面内联展开大面板。
- 列表操作列位置统一，分页交互统一。
- 表格工具栏只放操作和筛选控件，例如新增、批量删除、刷新、筛选；不要放说明性计数文本。
- 表格底部只放分页，不放总数、已选数量、状态说明等计数信息。
- 工作台页面优先实用和扫描效率，不做营销式视觉。
- 样式调整优先和现有工作台风格保持一致。
- 页面已有成熟交互范式时优先复用，不在相邻模块里做风格割裂的新方案。
- 拆分 Vue 业务模块时，共享 `provide/inject` key 必须使用运行时稳定的单例；跨模块 Symbol 优先使用 `Symbol.for(...)`，防止 HMR、重复模块实例或 `.ts` / `.js` 解析差异导致注入失败。
- 源码目录中若存在构建生成的 `.js` 影子文件，不能假设无扩展导入一定命中 `.ts`；共享上下文、API 单例、状态单例要额外确认运行时解析结果。

## 后端规则

- 服务之间按模块边界协作，不跨层偷逻辑。
- Worker 任务不能依赖前端瞬时状态。
- 外部调用必须设置 timeout、错误处理和可读日志。
- 错误场景不能伪装成功。

## 自动化规则

- 平台自动化优先稳定 DOM 选择器。
- 不使用固定分辨率点击作为正式方案。
- 页面结构变化时必须有日志、截图或诊断结果辅助排查。
- 不允许实现绕验证码、绕风控、异常规避逻辑。

## 模型与 Agent 规则

- 模型配置以页面配置为准，不依赖隐藏的 `.env` 默认值误导用户。
- Agent 只负责生成、总结、筛选和建议。
- Agent 不读取 Cookie，不直接调用发布器。
- Skill 只参与 prompt 约束，不直接写数据库或操作浏览器。
- 模型输出尽量结构化，避免下游每次解析都不一致。

## 安全规则

- 不在前端存储敏感 API Key 或 Cookie。
- 日志中不能泄露账号登录态、Cookie、完整 API Key。
- 发布和采集失败信息要可读，但不能附带敏感凭据。
