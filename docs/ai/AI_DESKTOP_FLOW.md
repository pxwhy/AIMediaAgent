# AI 桌面端流程

## 当前定位

当前项目还没有完全切到正式 Tauri 桌面分发形态，主要运行方式是“本地 API + H5 管理端 + 浏览器自动化 worker”。

桌面端的职责仍然是壳，不承载账号、采集、Agent、发布的核心业务逻辑。

## 当前启动流程

```text
执行启动脚本
  -> 启动 FastAPI
  -> 启动前端 H5
  -> 准备本地数据库和数据目录
  -> 用户在 H5 中发起登录、采集、发布、复盘等操作
  -> API 调用本地服务或子进程 worker
```

开发阶段常用脚本：

```bash
scripts/start_dev.sh
scripts/stop_dev.sh
```

## 登录流程

```text
H5 选择平台创建登录会话
  -> API 生成 LoginSession
  -> Playwright / worker 打开平台页面
  -> 用户手动完成登录
  -> H5 点击确认登录
  -> API 读取浏览器状态并落本地 state_path
  -> 创建或更新 Account
```

约束：

- 不绕验证码。
- 不伪造异常登录。
- 登录态只存在本地文件和账号模块内部。

## 作品同步流程

```text
H5 选择头条号账号点击同步作品
  -> API 调 account_work_service
  -> 启动 toutiao works worker
  -> 读取账号登录态
  -> 抓取作品标题、正文、时间、基础表现
  -> 去重写入 AccountWork
  -> 前端刷新作品列表
```

## 自动发布流程

```text
H5 从素材库创建草稿和发布任务
  -> API 写入 PublishTask
  -> 用户选择打开编辑器或自动发布
  -> publish_service 按平台分发到 worker
  -> worker 使用本地登录态打开平台页面
  -> 执行页面填写、图片处理、发布动作
  -> 保存运行日志、截图、平台返回
  -> API 回写任务状态和 PublishResult
  -> 前端查看诊断详情
```

平台注意事项：

- 头条号自动发布需要处理“投放广告赚收益”勾选。
- 小红书自动发布依赖稳定 DOM 选择器，尤其要考虑 Shadow DOM 结构。
- 正式方案优先 DOM 操作，不使用固定屏幕坐标作为长期方案。

## AI 流程

```text
H5 触发复盘 / 肖像 / 素材挑选
  -> API 读取账号作品、素材、Agent 配置、模型配置、Skills
  -> 组装 prompt
  -> 调用模型
  -> 解析结构化结果
  -> 持久化报告
  -> 前端弹框查看结果
```

## 后续正式桌面壳职责

- 管理窗口和托盘
- 启动和关闭本地服务
- 保存本地桌面级配置
- 统一升级入口

## 禁止事项

- 不在桌面壳里写发布策略。
- 不在桌面壳里写 Agent prompt 逻辑。
- 不在桌面壳里直接处理平台业务状态。
