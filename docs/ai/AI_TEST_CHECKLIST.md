# AI Test Checklist

## API

- `GET /health` 返回 `{"status":"ok"}`。
- 可以创建账号。
- 可以查询账号列表。
- 可以创建采集内容。
- 可以查询采集内容列表。
- 可以创建草稿。
- 可以查询草稿列表。
- 可以创建发布任务。
- 可以查询发布任务列表。
- 可以创建登录会话。
- 可以查询登录会话状态。
- 登录确认失败时返回明确错误。

## H5

- `npm run build` 通过。
- 首屏能显示仪表盘。
- API 不可用时显示连接失败状态。
- 移动端布局不溢出。
- 账号页可以手动新增账号。
- 账号页可以打开平台登录会话。
- 账号页可以确认登录并刷新账号列表。

## Worker

- Worker 可以独立启动。
- Worker 不阻塞 API。
- Worker 日志可查看。

## 模块边界

- Collector 不调用 Publisher。
- Agent 不读取 Cookie。
- Publisher 不调用模型。
- H5 不直接读写数据库。

## 安全

- `.env` 不提交。
- API Key 不出现在日志。
- Cookie 不返回给非必要接口。

## 发布自动化

- 所有外部请求必须设置 timeout。
- 平台页面变化时任务失败，而不是卡死。
- 验证码出现时进入人工处理，不绕过。
